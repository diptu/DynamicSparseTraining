# PhD application materials

The whole journey, packaged for admissions. A strong application isn't assembled in the final
month — it's the *narration* of a body of work that already exists. This repository is that work;
this folder turns it into SOPs, a research statement, a CV, and a per-program tracker.

> **Templates only.** Nothing here is filled with personal content on your behalf — these are
> structured prompts you complete. Drop real drafts in the same folders; keep the templates.

## Layout

```
applications/
├── README.md                      this file
├── cv/cv.md                       CV skeleton (academic format)
├── sop/TEMPLATE.md                statement of purpose — tailored per program
├── research-statement/TEMPLATE.md research statement — one, lightly adapted
├── programs/tracker.md            the application tracker table
└── programs/<school>.md           per-program notes (fit, faculty, deadlines)
```

## How this repo becomes an application

| Application asset | Sourced from |
|-------------------|--------------|
| Research statement | The arc in [`JOURNEY.md`](../../JOURNEY.md) + the paper's contribution |
| SOP "what I've done" | The paper, the [technical report](../reports/), the [log](../../paper/notes/log.md) |
| Evidence of independence | This repository itself — public, reproducible, sustained |
| Ability to communicate | The [blog posts](../blog/) |
| Knowledge of the field | The [literature review](../literature/) |
| Follow-through | Ten weekly [log](../../paper/notes/log.md) entries + git history |

## Working order

1. **Research statement first** — it's the intellectual core; the SOP borrows from it.
2. **CV in parallel** — it's mostly assembly.
3. **SOPs last, per program** — the same story aimed at specific faculty and fit.
4. **Tracker throughout** — deadlines, requirements, and status in one place.

## Standing advice baked into the templates

- Lead with the question and the result, not biography.
- Name specific faculty and *why* — tie to their actual work (the cold-email discipline from
  `Plan.md` Week 10 is the same muscle).
- Show taste through what you *cut*: a narrow, finished contribution beats a sprawling unfinished one.
